# VibeGuard 시연영상 대본 (자동 데모용)

이 대본은 `demo/demo.html` 자동 재생 화면의 해설이다. demo.html은 한국어 음성 해설을 브라우저 TTS로 자동 재생하므로, 직접 읽지 않아도 목소리가 들어간다(녹화할 때 시스템 소리가 함께 녹음되게만 하면 된다). 아래 문장은 그 해설 내용이며, 직접 읽거나 자막용으로 참고해도 된다. 1.25배속으로 진행되어 전체 약 1분 20초 내외다.

## 녹화 방법 (Windows 11)
1. `demo/demo.html` 을 더블클릭해 브라우저로 연다. F11 로 전체화면.
2. `Win + Alt + R` 로 화면 녹화를 시작한다. (윈도우 기본 기능, 설치 불필요)
3. 화면의 ▶ 를 클릭하면 자동으로 재생된다. 끝까지(약 1분 30초) 둔다.
4. `Win + Alt + R` 로 정지한다. 영상은 `내 PC \ 동영상 \ 캡처` 에 저장된다.
5. 유튜브에 업로드(공개/일부공개)하고 URL 을 결과보고서 '시연영상' 칸에 기재한다.

## 해설 (장면 순서대로, 괄호는 화면 표시 시간)

장면 1 — 인트로 (약 9초)
"요즘은 코딩을 몰라도 AI에게 맡겨 앱과 서비스를 만드는 시대입니다. 이걸 바이브코딩이라고 합니다. VibeGuard는 그렇게 만든 코드의 보안을 지켜 주는 도구입니다."

장면 2 — 왜 필요한가 (약 10초)
"문제는, AI가 만든 코드의 약 45퍼센트에 보안 취약점이 섞여 있다는 분석이 있다는 점입니다. 하드코딩된 키, 위험한 코드 실행, 가짜 패키지까지. 비전문가에게 보안은 너무 어렵습니다."

장면 3 — 스캔 (약 8초)
"사용법은 간단합니다. 검사할 폴더를 고르고 스캔만 누르면 됩니다."

장면 3-B — 실제 사용 화면 (약 9초)
"실제로는 이렇게, 브라우저에 결과 리포트가 그대로 뜹니다."

장면 4 — 탐지 결과 (약 13초)
"그러면 숨어 있던 문제들이 심각한 순서대로 정리되어 나옵니다. eval로 외부 입력을 실행하는 치명적 문제, TLS 인증서 검증을 꺼 버린 설정, 약한 MD5 암호까지. 각 항목마다 어디가, 왜 위험하고, 어떻게 고치는지 알려 줍니다."

장면 5 — 슬롭스쿼팅 (약 11초)
"가장 특징적인 기능입니다. AI가 지어낸 가짜 패키지, 슬롭스쿼팅까지 잡아냅니다. reqeusts나 expres처럼 유명 패키지와 한두 글자 다른 이름인데, 공격자가 이 이름을 미리 선점해 악성코드를 심습니다. 일반 보안 도구는 놓치는 위협입니다."

장면 6 — 점수 (약 9초)
"모든 결과는 0부터 100까지 보안 점수로 요약됩니다. 이 예제 앱은 0점, F등급이네요. 위험도를 한눈에 알 수 있습니다."

장면 7 — 누구나, 터미널 없이 (약 10초)
"코딩을 몰라도 됩니다. 실행파일을 받아 검사할 폴더를 끌어다 놓기만 하면, 브라우저에 리포트가 자동으로 열립니다. 터미널도, 파이썬도 필요 없습니다."

장면 8 — 마무리 (약 9초)
"규칙 54개, 6개 언어 지원, 외부 의존성은 0. AI가 만든 코드를 더 안전하게. VibeGuard였습니다. 감사합니다."

## 직접 시연하고 싶다면 (대안)
자동 데모 대신 실제 도구 화면을 녹화해도 된다. `python -m vibeguard gui` 를 실행하거나 실행파일에 폴더를 끌어다 놓고, `examples/vibe_coded_app` 를 스캔하는 과정을 캡처한다. 위 해설을 그대로 쓰면 된다. 슬롭스쿼팅(장면 5)을 가장 강조하고, 마지막에 저장소 주소(github.com/nohseongmin/VibeGuard)를 보여 준다.
