# VibeGuard 시연영상 대본 (목표 길이 약 2분 30초~3분)

대회 제출용 시연영상 녹화 대본입니다. 화면 녹화(예: OBS, 파워포인트 녹화, Windows Game Bar)로 터미널을 캡처하면서 아래 나레이션을 읽으면 됩니다. 유튜브 업로드 후 보고서 "시연영상" 칸에 URL을 기재하세요.

녹화 전 준비
- 터미널 글자 크기를 키워 가독성 확보(예: 16~18pt).
- 작업 폴더: VibeGuard 저장소 루트.
- 사전 설치: pip install -e .  (또는 python -m vibeguard 로 실행)
- 한글이 깨지지 않게 UTF-8 터미널 사용(Windows Terminal 권장. 본 도구는 출력 인코딩을 UTF-8로 고정함).

---

## 장면 1. 도입 — 문제 제기 (0:00 ~ 0:25)
화면: 발표 슬라이드 1장 또는 README 상단.
나레이션:
"요즘은 AI에게 코드를 맡기는 '바이브코딩'이 흔해졌습니다. 그런데 AI가 만든 코드에는 하드코딩된 비밀키, eval 같은 위험 함수, 취약한 암호가 그대로 들어가기 쉽고, 심지어 AI가 존재하지도 않는 패키지를 추천해 공격자가 그 이름으로 악성 패키지를 올리는 '슬롭스쿼팅' 위협까지 있습니다. VibeGuard는 이런 위험을 커밋 전에 자동으로 잡아주는 오픈소스 보안 가드레일입니다."

## 장면 2. 도구 소개 (0:25 ~ 0:40)
화면: 터미널에서 명령 입력.
입력:
```
vibeguard --version
vibeguard rules
```
나레이션:
"VibeGuard는 외부 의존성 없이 파이썬 표준 라이브러리만으로 동작합니다. 비밀키, 위험 실행, 인젝션, 웹 설정, 암호 등 여러 규칙군을 탑재하고 있습니다."

## 장면 3. 핵심 시연 — 취약 앱 스캔 (0:40 ~ 1:25)
화면: 데모 취약 앱 스캔.
입력:
```
vibeguard scan examples/vibe_coded_app
```
나레이션(결과를 가리키며):
"의도적으로 취약하게 만든 예제 앱을 스캔하면, 총 22건의 문제가 한국어로 보고됩니다. eval로 외부 입력을 실행하는 코드, TLS 인증서 검증을 꺼버린 설정, MD5 비밀번호 해시, 0.0.0.0 바인딩까지 — 각 항목마다 위치, 코드, 설명, 해결책, 그리고 CWE 번호가 함께 나옵니다."

## 장면 3-B. GUI로 한눈에 보기 (옵션)
화면: 터미널에서 `vibeguard gui` 실행 → 자동으로 열린 브라우저.
입력:
```
vibeguard gui
```
나레이션:
"터미널이 익숙하지 않은 사용자를 위해 브라우저 GUI도 제공합니다. 같은 스캔 결과가 보안 점수 링, 심각도별 칩 필터, 취약점 카드로 한눈에 정리되고, 슬롭스쿼팅 항목에는 '공급망' 태그가 붙습니다. 외부 프레임워크 없이 표준 라이브러리만으로 동작합니다."

## 장면 4. 차별점 — 슬롭스쿼팅 탐지 (1:25 ~ 1:55)
화면: 출력에서 VG-SLOP 항목 강조(스크롤 또는 확대).
나레이션:
"가장 특징적인 기능입니다. requirements.txt의 'reqeusts'와 package.json의 'expres'를 보세요. 각각 인기 패키지인 'requests', 'express'와 철자가 한두 글자 다른 환각·오타 패키지입니다. VibeGuard는 편집거리 분석과 레지스트리 조회로 이런 공급망 위협을 잡아냅니다. 일반 보안 도구에는 드문 기능입니다."

## 장면 5. 점수·출력 형식 (1:55 ~ 2:15)
화면: 요약 점수 + JSON 출력.
입력:
```
vibeguard scan examples/vibe_coded_app --format json
```
나레이션:
"전체 결과는 0에서 100 사이 보안 점수와 등급으로 요약됩니다. 이 앱은 0점, F등급이네요. JSON 형식으로도 출력되어 CI 파이프라인이나 에디터에 그대로 연동할 수 있습니다."

## 장면 6. git 커밋 가드레일 (2:15 ~ 2:40)
화면: 훅 설치 후 커밋 시도.
입력:
```
vibeguard init-hooks
git add .
git commit -m "test"
```
나레이션:
"init-hooks 한 번이면 git pre-commit 훅이 설치되어, 치명적 문제가 있는 코드는 커밋 자체가 막힙니다. 바이브코딩 흐름에 그대로 끼워 넣는 안전벨트인 셈입니다."

## 장면 7. 마무리 (2:40 ~ 3:00)
화면: README 또는 GitHub 저장소.
나레이션:
"VibeGuard는 테스트 36개를 모두 통과했고, 런타임 의존성이 전혀 없어 도구 자체의 공급망 위험도 0입니다. 누구나 무료로 쓰고 규칙을 확장할 수 있는 오픈소스로 공개되어 있습니다. AI가 만든 코드를 더 안전하게, VibeGuard였습니다. 감사합니다."
