1. 개발배경 및 목적

1.1 배경

AI 코딩 어시스턴트(Cursor, Claude Code, GitHub Copilot, v0, Bolt, Lovable 등)로
자연어 지시를 주고 받아 나온 코드를 깊이 검토하지 않고 그대로 실행, 배포하는
개발 방식이 빠르게 확산되고 있다. 이를 흔히 바이브코딩(vibe coding)이라 부른다.
생산성은 비약적으로 올랐지만, 사람이 한 줄씩 읽고 검증하던 단계가 사라지면서
보안 결함이 코드에 그대로 섞여 들어간다.

이것은 일화가 아니라 측정된 현상이다.
- AI가 생성한 코드의 약 45퍼센트가 보안 취약점을 포함한다는 분석이 보고되었다
  (2025년 보안 업계 보고서).
- 한 분석은 AI 에이전트가 출시하는 동작 기능 10개당 약 9개가 악용 가능한 보안
  취약점을 동반한다고 표현했다.
- 자연어로 반복해 다듬는 과정에서 인증 로직이 조용히 약해지거나 일부 제거되고,
  엔드포인트 권한 검사가 누락되며, 화면을 바꾸는 사이 보호되지 않은 API가 그대로
  열려 있는 일이 반복된다. "바이브" 방식으로 다섯 번 반복하면 초기 버전보다 치명적
  취약점이 약 37퍼센트 더 늘어난다는 보고도 있다. 즉 반복할수록 위험이 누적된다.
- 여러 AI 코딩 도구로 만든 운영 수준 애플리케이션을 점검한 연구에서, 검사한
  모든 앱이 보안 헤더 설정이 없었고 서버측 요청 위조(SSRF)에 취약했다는 결과가
  나왔다.
- AI가 만든 코드의 약 20퍼센트가 실제로 존재하지 않는 패키지를 참조한다. 공격자가
  이렇게 자주 환각되는 이름을 미리 레지스트리에 선점 등록해 악성코드를 심어 두는
  슬롭스쿼팅(slopsquatting)이라는 새로운 공급망 공격으로 이어진다.

특히 반복 프롬프팅이 위험을 누적시킨다는 점은, 보안 점검이 개발이 끝난 뒤 한 번이
아니라 코드가 바뀌는 매 순간 루프 안에서 이루어져야 함을 뜻한다. VibeGuard가 git
커밋과 AI 에이전트의 코드 수정 직후에 자동 실행되도록 설계된 이유가 여기에 있다.

1.2 문제 정의

바이브코딩을 하는 사람은 대체로 보안 전문가가 아니다. 학생, 1인 개발자,
비개발 직군의 사내 제작자처럼 "일단 동작하는 결과물"을 빠르게 만드는 데 집중하는
사용자가 많다. 그런데 기존 보안 정적 분석(SAST) 도구는 보안 전문가를 대상으로
설계되어, 설치와 설정이 무겁고, 결과가 전문 용어로 가득하며, 무엇을 어떻게
고쳐야 하는지 평이하게 알려주지 않는다. 또한 AI 시대의 새로운 위협인
슬롭스쿼팅을 다루는 도구는 드물다.

즉, 가장 보안 도움이 필요한 사용자가 가장 도구를 쓰기 어려운 역설이 존재한다.

1.3 목적

VibeGuard는 바이브코딩의 흐름을 끊지 않으면서 코드의 보안 문제를 잡아내고,
비보안 전문가도 이해할 수 있는 언어로 "무엇이, 왜 위험하고, 어떻게 고치는지"를
알려주는 가벼운 오픈소스 가드레일을 목표로 한다. 구체적으로 다음을 지향한다.
- 빠르고 가볍게: 설치 없이 한 줄 명령으로 실행, 런타임 의존성 0.
- 쉽게: 모든 탐지 결과에 한국어 설명과 수정 가이드를 제공.
- 루프 안에서: git 커밋과 AI 에이전트의 코드 수정 직후 자동 실행.
- AI 시대 위협 대응: 슬롭스쿼팅, 오타스쿼팅 등 공급망 위협을 기본 탐지.


2. 개발환경 및 개발언어

2.1 개발 언어 및 실행 환경
- 개발 언어: Python 3 (3.8 이상 지원)
- 실행 환경: Windows, macOS, Linux (운영체제 독립적)
- 런타임 의존성: 없음. 파이썬 표준 라이브러리(re, os, urllib, json, argparse 등)만
  사용한다. 이는 보안 도구 자신이 외부 패키지 공급망 위험을 끌어들이지 않도록 한
  의도적 설계다.

2.2 분석 대상
- Python(.py), JavaScript(.js, .jsx, .mjs, .cjs), TypeScript(.ts, .tsx)
- 시크릿(비밀키) 탐지 규칙은 모든 텍스트 파일에 적용
- 의존성 파일: requirements.txt, package.json

2.3 개발 및 품질 관리 도구
- 테스트: pytest (단위 테스트 30여 건)
- 패키징: setuptools, pyproject.toml (pip install 로 vibeguard 명령 등록)
- 버전관리: git, git pre-commit 훅 연동

2.4 활용한 공개SW
- Python 및 표준 라이브러리
- 패키지 존재 여부 대조 대상으로 PyPI, npm 공개 레지스트리 API
- git(훅 연동), pytest(테스트)
- 산출물은 MIT 라이선스로 공개하여 누구나 자유롭게 사용, 수정, 재배포 가능


3. 시스템 구성 및 아키텍처

3.1 전체 구조

사용자 또는 자동화 훅이 명령을 실행하면, 스캐너가 대상 폴더를 순회하며 각 파일에
규칙을 적용하고, 동시에 의존성을 모아 레지스트리와 대조한다. 탐지 결과(Finding)는
점수 계산기와 리포터를 거쳐 사람이 읽기 쉬운 형태로 출력된다.

  [실행 진입점]
    CLI (vibeguard scan / rules / init-hooks)
        |
        v
  [스캐너 엔진]  scanner.py
    - 폴더 순회(산출물 디렉터리 제외)
    - 파일별로 적용 가능한 규칙만 선별 적용
        |
        +---> [규칙 엔진]   rules/  (secrets, dangerous, injection, web, crypto)
        |
        +---> [공급망 검사] slopsquat.py
        |        - import/의존성 추출
        |        - 표준 라이브러리 제외
        |        - PyPI/npm 레지스트리 존재 확인
        |        - 유명 패키지 오타스쿼팅(편집거리) 검사
        |
        v
  [탐지 결과 모델]  finding.py  (규칙ID, 심각도, 위치, 설명, 해결, CWE)
        |
        +---> [점수 계산]  score.py  (0~100 점, A~F 등급, 한 줄 평)
        |
        v
  [리포터]  reporter.py  (terminal / json / markdown)
        |
        v
  [통합 훅]  hooks/  (git pre-commit, Claude Code 등 AI 에이전트 PostToolUse)

3.2 모듈 설명
- finding.py: 탐지 1건을 표현하는 데이터 모델과 심각도(정보/낮음/중간/높음/치명적).
- rules/base.py: 규칙 추상화와 전역 레지스트리, 오탐을 줄이는 placeholder 필터.
- rules/secrets.py: 하드코딩된 API 키, 토큰, 개인키, 비밀번호 탐지.
- rules/dangerous.py: eval, exec, shell=True, pickle, yaml.load 등 위험 실행.
- rules/injection.py: 문자열 조립 SQL, 템플릿 SQL, MongoDB $where.
- rules/web.py: 디버그 모드, 전체 허용 CORS, TLS 검증 비활성화, XSS 패턴.
- rules/crypto.py: MD5/SHA1, 예측 가능한 난수, DES/ECB 등 약한 암호화.
- slopsquat.py: 슬롭스쿼팅, 오타스쿼팅 탐지(레지스트리 대조 + 편집거리).
- scanner.py: 파일 수집과 순회, 라인 단위 무시 주석 처리.
- score.py: 바이브 보안 점수와 등급.
- reporter.py: 터미널, JSON, 마크다운 출력.
- cli.py: 명령행 인터페이스와 CI 친화적 종료 코드.
- hooks/: git 및 AI 에이전트 훅 설치.

3.3 설계 원칙
- 규칙은 데이터로 표현해 새 규칙 추가가 쉽다(확장성).
- 라인 단위 무시 주석(vibeguard: ignore)과 placeholder 필터로 오탐을 줄인다.
- 네트워크가 없으면 오프라인 모드로 동작(레지스트리 조회를 건너뛰고 휴리스틱만).


4. 주요기능

4.1 AI 코드 특유의 취약 패턴 정적 탐지
여섯 개 분류에 걸쳐 40여 개 규칙을 탑재한다.
- 시크릿: OpenAI, Anthropic, AWS, GitHub, Stripe, Slack 키, 개인키(PEM),
  평문 비밀번호, DB 접속 URL 내 비밀번호.
- 위험한 실행: eval/exec, subprocess shell=True, os.system 문자열 포매팅,
  pickle 역직렬화, yaml.load, JavaScript eval/new Function, child_process.
- 인젝션: f-string/문자열결합/템플릿문자열로 만든 SQL, MongoDB $where.
- 웹 설정 결함: 디버그 모드, 전체 허용 CORS, TLS 인증서 검증 비활성화,
  innerHTML/dangerouslySetInnerHTML(XSS).
- 약한 암호화: MD5/SHA1, 예측 가능한 난수로 토큰 생성, Math.random, DES/ECB.
각 결과에는 규칙 식별자, 심각도, 파일과 라인 위치, 문제 코드, 평이한 설명,
구체적 수정 방법, 관련 CWE 번호가 포함된다.

4.2 슬롭스쿼팅 및 오타스쿼팅 탐지 (핵심 차별점)
소스코드의 import 와 requirements.txt, package.json 의 의존성을 모은 뒤,
표준 라이브러리를 제외한 외부 패키지가 실제 PyPI 또는 npm 레지스트리에 존재하는지
확인한다. 존재하지 않으면 AI 가 지어낸 환각 패키지일 수 있으므로 경고한다(공격자가
그 이름을 선점하면 설치 즉시 감염될 수 있다). 또한 유명 패키지와 철자가 매우 비슷한
이름(편집거리 1~2)을 오타스쿼팅 후보로 함께 경고한다. 이 기능은 AI 시대의 새로운
공급망 위협을 겨냥한 것으로, 일반 SAST 도구와 구별되는 VibeGuard의 핵심 가치다.

4.3 비전문가를 위한 설명과 수정 가이드
모든 탐지 항목에 "설명(왜 위험한가)"과 "해결(어떻게 고치는가)"을 한국어로 제공한다.
보안 용어를 모르는 사용자도 바로 이해하고 조치할 수 있도록 작성했다.

4.4 바이브 보안 점수
탐지 결과의 심각도를 가중 합산해 0~100점의 보안 점수와 A~F 등급, 한 줄 총평을
제시한다. 사용자가 자신의 프로젝트 상태를 직관적으로 파악하고 개선 동기를 갖도록
돕는다.

4.5 바이브코딩 루프 자동 결합
- git pre-commit 훅: 커밋 직전 자동 스캔, 높음 이상 발견 시 커밋 차단(우회 가능).
- AI 에이전트 훅: Claude Code 등에서 파일 수정 직후 자동 점검하도록 설정 예시 제공.
  에이전트가 결과를 보고 스스로 코드를 교정하는 폐루프(self-correction)를 만든다.

4.6 다양한 출력 형식
- 터미널: 사람이 바로 읽는 콘솔 리포트(색상 옵션).
- JSON: CI 파이프라인과 에디터 연동용 기계 판독 형식.
- 마크다운: PR과 이슈, 문서 첨부용.

4.7 오탐 억제
- 줄 끝 vibeguard: ignore 주석으로 특정 줄 제외.
- your-api-key, example, xxxx 등 placeholder 값은 시크릿 규칙에서 자동 제외.
- node_modules, .venv, dist 등 산출물 디렉터리는 스캔 대상에서 제외.

6. 기대효과 및 활용분야

6.1 기대효과
- 보안 비전문가가 만든 코드의 사고를 예방한다. 특히 코드에 박힌 비밀키 유출과
  슬롭스쿼팅 공급망 감염처럼 피해가 큰 사고를 조기에 차단한다.
- 보안 지식이 없어도 결과 설명을 읽으며 자연스럽게 안전한 코딩을 학습하게 된다.
- 사람이 검토하지 못하는 AI 생성 코드의 빈틈을, 같은 속도의 자동 점검으로 메운다.

6.2 활용분야
- 1인 개발자, 학생, 인디 해커, 비개발 직군 제작자의 개인 안전망.
- 교육 현장: 안전한 코딩 학습 보조 도구.
- 팀과 기업: CI 파이프라인과 PR 게이트에 결합해 배포 전 차단.
- AI 에이전트 개발 환경: 에이전트가 점검 결과를 받아 스스로 교정하는 루프 구성.
- 오픈소스 생태계: MIT 공개로 누구나 규칙을 추가, 개선, 재배포 가능.

6.3 공개SW로서의 의의
VibeGuard는 그 자체가 외부 의존성 없는 오픈소스이며, AI 시대에 폭증하는 코드의
안전을 공개SW 방식으로 함께 끌어올리는 것을 지향한다. 규칙을 데이터로 표현해
커뮤니티가 새로운 위협 패턴을 손쉽게 기여할 수 있도록 설계했다.


[부록] 데모 및 검증

저장소의 examples/vibe_coded_app 폴더에는 AI 에게 "간단한 메모 저장 API 를
만들어줘"라고 지시했을 때 나올 법한, 빠르게 동작하지만 보안 결함이 가득한 데모 앱이
들어 있다(모든 키 값은 가짜다). 이 폴더를 스캔하면 하드코딩된 키, SQL 인젝션,
명령어 주입, TLS 검증 비활성화, 약한 해시, 예측 가능한 토큰, 디버그 모드, 그리고
requirements.txt 의 환각 패키지와 오타스쿼팅 후보까지 한 번에 탐지되어, 보안 점수가
어떻게 산출되는지 확인할 수 있다. tests 폴더의 단위 테스트로 각 규칙의 동작을
검증한다.
